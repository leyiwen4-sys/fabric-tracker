# 布料记录工具 - 四个增强功能 设计规格说明

**日期：** 2026-06-03
**状态：** 已批准

---

## 1. 项目概述

为现有布料记录工具新增四个功能：搜索、统计面板、用途状态标记、多图支持。

## 2. 数据模型变更

### fabrics 表新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `status` | TEXT | `'idle'` 闲置 / `'used'` 已用 / `'empty'` 已用完，默认 `'idle'` |
| `photos` | TEXT | JSON 数组，最多 3 张。如 `'["/uploads/a.jpg","/uploads/b.jpg"]'` |

### 迁移策略

- 原 `photo_path` 字段保留兼容，新逻辑优先读 `photos`
- 如果 `photos` 为空但 `photo_path` 有值，自动视为单图
- 新增布料时统一写入 `photos`，同时填充 `photo_path` 为第一张

## 3. 功能 A：搜索

### 首页搜索栏

header 下方新增搜索栏：

```
┌──────────────────────────┐
│ 🔍 搜索布料名称、店铺...    │
└──────────────────────────┘
```

- 纯文本搜索，匹配 name、store、notes 三个字段
- 使用 SQL LIKE 查询（`WHERE (name LIKE ? OR store LIKE ? OR notes LIKE ?)`）
- API `/api/fabrics` 新增 `?search=` 查询参数

## 4. 功能 B：统计面板

### 新增页面 `/stats`

首页 header 中新增 📊 入口按钮。

统计内容（纯文字 + CSS，不引入图表库）：

- 📦 总布料数
- 💰 总花费（所有布料 price 之和）
- 📊 按类型分布（如：棉 ██████ 5 块，麻 ██ 2 块 ...）
- 📋 按状态分布（闲置 X / 已用 X / 已用完 X）

### 新增 API

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/stats` | 返回当前用户的统计数据 |

## 5. 功能 C：用途状态标记

- **详情页：** 标签行增加状态标签，点击可在三种状态间切换
- **首页卡片：** 右上角小圆点角标（🟢闲置 / 🟡已用 / ⚪已用完）
- **表单页：** 增加状态选择（默认"闲置"）

### 状态值

| 值 | 显示 | 颜色 |
|------|------|------|
| `idle` | 闲置 | 绿色 |
| `used` | 已用 | 黄色 |
| `empty` | 已用完 | 灰色 |

## 6. 功能 D：多图（最多 3 张）

### 表单页
- 上传区支持选多张（`<input multiple>`，最多 3 张）
- 已选照片显示缩略图预览，可点击 × 删除单张
- 编辑时显示已有照片，可删除/新增（总不超过 3 张）

### 详情页
- 主图区显示当前选中的照片（大图）
- 下方缩略图横排，点击切换主图
- 单图时不显示缩略图条

### 首页卡片
- 显示第一张照片

### API 变更
- POST/PUT `/api/fabrics`：接受多个 photo 文件（`photo_0`, `photo_1`, `photo_2`），服务端合并为 JSON 数组存入 `photos` 字段

## 7. 路由更新

| 路由 | 说明 |
|------|------|
| `/` | 首页（加搜索栏、卡片加状态角标） |
| `/fabrics/[id]` | 详情（多图切换、状态标签点击切换） |
| `/fabrics/new` | 添加（多图上传、状态选择） |
| `/fabrics/[id]/edit` | 编辑（多图编辑、状态选择） |
| `/stats` | 统计页（新增） |
| `/api/stats` | 统计 API（新增） |

## 8. 不在范围（YAGNI）

- 图片拖拽排序
- 类型筛选下拉
- 图表库（纯 CSS 实现）
- 统计数据导出
- 图片编辑/裁剪
