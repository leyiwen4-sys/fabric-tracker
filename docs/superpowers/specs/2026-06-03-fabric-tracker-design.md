# 布料记录工具 - 设计规格说明

**日期：** 2026-06-03
**状态：** 已批准

---

## 1. 项目概述

一个移动端网页应用，用于记录和浏览个人购买的布料。核心体验是像相册一样翻看布料收藏。

## 2. 技术栈

| 层级 | 选型 |
|------|------|
| 框架 | Next.js（全栈一体） |
| 前端 | React + 移动端优先 CSS |
| 后端 API | Next.js API Routes |
| 数据库 | SQLite（零配置，文件存储） |
| 照片存储 | 本地文件系统 `public/uploads/` |
| ORM | better-sqlite3（同步、轻量） |

## 3. 数据模型

**表：`fabrics`**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PK, AUTOINCREMENT | 自增主键 |
| `name` | TEXT | NOT NULL | 布料名称 |
| `type` | TEXT | NOT NULL | 类型：棉/麻/丝/毛/化纤/混纺/其他 |
| `width` | REAL | - | 幅宽（cm） |
| `unit` | TEXT | NOT NULL | 单位：米 / 码 |
| `price` | REAL | - | 单价（元/单位） |
| `store` | TEXT | - | 购买店铺 |
| `purchase_date` | TEXT | - | 购买日期（ISO 8601） |
| `photo_path` | TEXT | - | 照片文件路径 |
| `notes` | TEXT | - | 备注 |
| `created_at` | TEXT | NOT NULL | 创建时间（ISO 8601） |
| `updated_at` | TEXT | NOT NULL | 更新时间（ISO 8601） |

共 9 个业务字段 + 2 个时间戳字段。

## 4. 页面与路由

| 路由 | 页面 | 说明 |
|------|------|------|
| `/` | 相册首页 | 双列卡片网格，右下角悬浮 `+` 按钮 |
| `/fabrics/[id]` | 布料详情 | 大图 + 标签 + 详细信息 + 编辑/删除 |
| `/fabrics/new` | 添加布料 | 表单页，含拍照/上传区域 |
| `/fabrics/[id]/edit` | 编辑布料 | 与添加页共用表单组件，预填数据 |

## 5. API 设计

所有 API 返回统一格式：`{ success: boolean, data?: any, error?: string }`

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/fabrics` | 列表查询，支持 `?type=&sort=created_at_desc` |
| `GET` | `/api/fabrics/[id]` | 单条详情 |
| `POST` | `/api/fabrics` | 新增（multipart/form-data，含照片） |
| `PUT` | `/api/fabrics/[id]` | 更新（multipart/form-data） |
| `DELETE` | `/api/fabrics/[id]` | 删除（同时删除关联照片文件） |

## 6. UI 设计（移动端优先）

### 6.1 相册首页
- 双列网格布局，每张卡片：照片区域 + 名称 + 类型·幅宽·单价
- 顶部标题栏 + 布料总数
- 右下角悬浮操作按钮（FAB），点击进入添加页

### 6.2 布料详情
- 顶部返回按钮 + 布料名称
- 全宽大图展示
- 标签行：类型、幅宽、单价
- 信息列表：店铺、日期、备注
- 底部操作按钮：编辑（主按钮）、删除（次按钮/危险色）

### 6.3 添加/编辑表单
- 顶部返回按钮 + 标题
- 大块虚线拍照/上传区域
- 表单字段单列排列，部分字段（类型+幅宽、单位+单价）同行
- 底部保存按钮
- 编辑时预填已有数据

## 7. 错误处理

- **前端校验：** 名称必填，提交前即时校验
- **后端校验：** 必填字段检查，返回 400 + 具体错误描述
- **文件上传：** 限制 10MB，仅允许图片格式（jpg/png/webp）
- **网络异常：** Toast 提示用户重试
- **删除保护：** 弹出确认对话框，防止误删

## 9. 不在范围（YAGNI）

- 用户登录/多用户
- 搜索和筛选（后续迭代）
- 分类/标签系统
- 数据导出/导入
- 库存消耗跟踪
- 多设备同步
